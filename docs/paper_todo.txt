This is the "Developer’s Manual" for implementing the VABL (Variational Attention-based Belief Learning) framework from scratch. This guide breaks down the abstract math into concrete tensor operations, data structures, and environment-specific logic.Phase 1: The Core Architecture (The "Brain")You are building a single neural network class (e.g., VABLAgent) that is shared across all agents (Parameter Sharing). However, each agent maintains its own hidden state (hidden_state) during a rollout.1. InputsAt every timestep $t$, for agent $i$, the network receives:Local Observation ($o_t^i$): A vector describing what the agent sees (e.g., 64-dim float).Teammate Actions ($a_{t-1}^{-i}$): A list or tensor of the previous actions taken by all other agents.Visibility Mask ($m_t$): A binary vector (size $N-1$) indicating which teammates are currently visible.Previous Belief ($b_{t-1}^i$): The hidden state from the previous step (e.g., 128-dim float).2. The Modules (PyTorch Implementation Details)Observation Encoder (phi_net):Input: obs_dim (e.g., 42).Output: embed_dim (e.g., 64).Layer: nn.Sequential(nn.Linear, nn.ReLU, nn.Linear, nn.ReLU).Purpose: Compress the raw visual/feature input into a workable embedding.Action Encoder (psi_net):Input: action_dim (one-hot encoded) or scalar index.Output: embed_dim (e.g., 64).Layer: nn.Embedding (if discrete) or nn.Linear (if continuous/one-hot).Purpose: Turn "Agent 2 moved Up" into a rich vector representation.Attention Mechanism (The "Social" Layer):*Query ($Q$): The previous belief $b_{t-1}^i$ (passed through a linear layer W_q).Keys ($K$) & Values ($V$): The encoded teammate actions $\psi(a_{t-1}^{-i})$ (passed through W_k and W_v).Operation: scores = torch.matmul(Q, K.transpose()) / sqrt(d_k).Masking (Crucial): Before the softmax, apply scores.masked_fill_(mask == 0, -1e9). This ensures invisible agents have 0 attention weight.Output: context = torch.matmul(softmax(scores), V).Recurrent Unit (The "Memory"):Input: Concatenation of [obs_embedding, context_vector].Layer: nn.GRUCell(input_size=embed_dim*2, hidden_size=hidden_dim).Output: New belief $b_t^i$.Heads:Policy Head: nn.Linear(hidden_dim, action_dim) $\rightarrow$ Output logits for my action.Auxiliary Head: nn.Linear(hidden_dim, action_dim * (N-1)) $\rightarrow$ Output predictions for every other agent's next action.Phase 2: The Forward Pass (Step-by-Step)Here is the exact data flow for one timestep in code logic:Embed: Pass obs through phi_net to get h_obs.Prepare Social Inputs:Take prev_actions of all agents.If $t=0$, replace prev_actions with a learned start_token parameter.Pass prev_actions through psi_net to get h_actions.Attend:Generate Q from hidden_state.Generate K, V from h_actions.Compute Attention with the visibility_mask.Result is context_vector (which captures: "What did my meaningful teammates just do?").Update Belief:rnn_input = cat(h_obs, context_vector).new_hidden = gru(rnn_input, hidden_state).Predict:policy_logits = policy_head(new_hidden).aux_logits = aux_head(new_hidden) (Shape: [batch, num_teammates, action_dim]).Phase 3: The Loss Calculation (The "Learning")This is where VABL differs from standard PPO/MAPPO. You have a compound loss function.$$L_{total} = L_{RL} + \lambda \cdot L_{Aux}$$1. The RL Loss ($L_{RL}$)Use standard MAPPO (PPO with centralized value function) or QMIX.Inputs: policy_logits, actions_taken, advantages, values, returns.Calculation: Standard policy gradient clipping or TD-error.2. The Auxiliary Loss ($L_{Aux}$)Goal: Minimize prediction error of teammate actions.Inputs: aux_logits (from time $t$), true_next_actions (from time $t+1$), next_visibility_masks (at time $t+1$).Logic:You predicted Agent 2's action at time $t$ using belief $b_t$.At time $t+1$, you verify: "Was Agent 2 visible? And did they do what I expected?"Code Implementation:Python# aux_logits: [batch, T, num_agents, num_others, action_dim]
# target_actions: [batch, T, num_agents, num_others] (indices of actual actions)
# mask: [batch, T, num_agents, num_others] (1 if visible, 0 if not)

loss_fn = nn.CrossEntropyLoss(reduction='none')
loss = loss_fn(aux_logits.view(-1, action_dim), target_actions.view(-1))

# Apply Mask: Only punish wrong predictions for VISIBLE teammates
loss = loss * mask.view(-1)

# Average over non-masked elements
aux_loss = loss.sum() / (mask.sum() + 1e-8)
Phase 4: Environment-Specific Implementation1. Overcooked (The "Coordination" Test)Why here? It tests if agents can infer "Goal: Delivery" vs "Goal: Chopping" without words.Observation: Grid view (centered on agent).Action Space: [Up, Down, Left, Right, Interact, Stay].Visibility: Often full visibility, but you can simulate partiality (fog of war).Crucial Implementation Detail:The Feature Encoder: Should specific channels for "Onion locations", "Pot locations", and "Teammate locations".The Latent Factor: The belief needs to encode sub-tasks. If Agent A moves toward an onion, Agent B's aux head should predict Agent A will continue moving toward the onion. This forces Agent B's belief to realize "A is fetching ingredients."2. StarCraft II (SMAC / SMACv2) (The "Focus Fire" Test)Why here? Agents must focus fire on the same enemy to win (implicit coordination).Visibility: Strictly partial (limited sight range).Action Space: [Move(Direction), Attack(Enemy_ID), Stop, Heal].Implementation Nuances:Variable Agent Counts: In SMAC, agents die. Your prev_actions tensor will have dead agents.Dead Agent Masking: The visibility_mask MUST be 0 for dead agents. If you attend to a dead agent (who sends a "No-Op" action), you pollute the context.Relative Positions: The obs contains relative $(x,y)$ of allies. VABL uses the action history, but the spatial position is in the obs. The GRU must fuse "Teammate is at (5,5)" (from obs) with "Teammate just Attacked Unit 3" (from attention) to infer "Teammate is aggressively engaging Unit 3."3. SMAC v2 (The "Generalization" Test)Difference: Start positions and unit types are randomized.Implementation:Your encoders need to be robust to different unit types (e.g., Marines vs. Stalkers).You might need a Type Embedding added to the input of the phi_net so the agent knows what kind of unit it is observing.Phase 5: Implementation Checklist (Zero to Hero)Define the Shared Policy Network: Create the VABL_Network class with the Encoder $\to$ Attention $\to$ GRU $\to$ Heads flow.Build the Rollout Buffer: Ensure it stores obs, actions, rewards, dones, AND visibility_masks.The "Bootstrap" Step: In your training loop, handle the bootstrap value for the last step (standard PPO stuff), but ALSO handle the "next action" target for the Aux loss (shift the action tensor by 1).Integration:Start with a simple environment (e.g., Overcooked).Verify aux_loss decreases. If aux_loss stays high ($\ln(\text{num\_actions})$), your model isn't learning to predict teammates (Theory of Mind failed).Once aux_loss drops, check if rewards increase.Ablation (Scientific Rigor):Run a version with $\lambda = 0$ (Aux loss off). This is your baseline (essentially a recurrent attention agent).The difference in performance is your "Implicit Coordination Gain."This breakdown covers everything from the tensor shapes to the high-level logic needed to reproduce the results in the paper.
